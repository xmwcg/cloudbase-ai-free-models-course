## 1. 背景与目标（Why）

当前 `/ai-chat` 已能对话（hy3 / hy3-preview）和文生图（HY-Image-3.0-Plus），但：

- 图生图能力 `HY-Image-v3.0-I2I-ToB-v1.0.1` **后端未实现、前端无入口**。
- 图像模式模型写死，用户无法在「文生图 / 图生图」之间切换。
- 模型无统一清单与免费标识，调用链路对额度不透明。

**目标（Goals）**

1. 后端 `ai-image` 支持 I2I 模型（接收底图 base64）。
2. 前端在「模型选择」中暴露 4 个免费模型，文本 2 + 图像 2。
3. 图生图模式下出现「上传底图」控件。
4. 全链路走小程序成长计划免费额度，无额外计费风险。

## 5. 后端云函数改动

### 5.1 `ai-chat`（文本，最小改动）
- 增加 **模型白名单**：只允许 `hy3` / `hy3-preview`，其余返回 400。
- 默认 `model = "hy3"` 保持不变。
- 现有 `stream: false` 逻辑保留（后续可加 SSE，本期不做）。

### 5.2 `ai-image`（图像，新增 I2I）— 重点
新增对 `HY-Image-v3.0-I2I-ToB-v1.0.1` 的支持：

- 入参新增 `image`（`string`，base64 或 data URL）。
- 模型白名单：`HY-Image-3.0-Plus-4090-Tob-v1.0` 与 `HY-Image-v3.0-I2I-ToB-v1.0.1`。
- 当 `model` 为 I2I 且缺 `image` → 400 `image is required for I2I`。
- 调用时把 `image` 透传给 `generateImage`（字段名以 CloudBase AI SDK 为准，见 §9 风险项；
  若 SDK 要求去前缀 base64，则在函数内 `image.split(",")[1]` 处理）。

```js
const res = await imageModel.generateImage({
  model,
  prompt,
  size,
  enable_thinking: { value: false },
  ...(isI2I ? { image } : {}),   // I2I 才带底图
});
```

### 5.3 免费标识
- 两个函数响应体已含 `model` 字段；前端据此显示「Free」角标。
- 不需要鉴权，因为走小程序计划免费额度（公开调用由 CORS + 额度兜底）。

## 6. 前端 `chat.html` 改动

### 6.1 模型选择统一
- 对话模式：下拉绑定 `MODELS.text`（已是 hy3 / hy3-preview，保留）。
- 图像模式：新增 **Image Model 下拉**绑定 `MODELS.image`（文生图 / 图生图）。

### 6.2 图生图上传控件
- 当图像模型 = I2I 时，显示 `id="i2iUpload"`（`<input type="file" accept="image/*">`）。
- `generateImage()` 内：若 I2I，用 `FileReader` 读底图为 base64，随请求体发送 `image`。

```js
function generateImage() {
  const prompt = imageInput.value.trim();
  const model = imageModelSelect.value;
  const isI2I = model.includes("I2I");
  if (!prompt || isGenerating) return;
  let imageB64 = null;
  if (isI2I) {
    const f = document.getElementById("i2iFile").files[0];
    if (!f) { alert("图生图请先上传底图"); return; }
    imageB64 = await fileToBase64(f);   // data URL
  }
  // ...fetch(IMAGE_API, { body: JSON.stringify({ prompt, size: imageSize, model, image: imageB64 }) })
}
```

- 侧边栏 footer 保持「Powered by Mini Program Growth Plan Free Quota」。

## 9. 风险与注意

1. **I2I SDK 字段名**：`hunyuan-image` 的 `generateImage` 对图生图的底图参数名需以 CloudBase AI SDK
   官方文档为准（可能 `image` / `image_base64` / `image_url`）。部署前用一次真实调用验证，
   若函数报错「unexpected field」，按报错调整透传字段。
2. **base64 体积**：底图建议前端压缩到 ≤ 2MB 再上传，避免云函数超时 / 请求体过大。
3. **cloudbaserc.json**：当前只声明 `ai-chat`，务必补充 `ai-image` 与 `ai-page` 否则部署不全。
4. **CORS**：两个函数已带 `Access-Control-Allow-Origin: *`，浏览器直连无需代理；若后续加域名白名单需同步。
5. **额度监控**：小程序成长计划免费额度（文本 10 亿 token / 图像 10 万张）用尽后调用会失败，
   建议在 CloudBase 控制台设置用量告警。
6. **公开暴露**：云函数地址公开可被刷，依赖额度兜底；如需防护可在函数层加简单签名或 Referer 校验。
